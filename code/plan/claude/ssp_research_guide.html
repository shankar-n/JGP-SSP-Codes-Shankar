Task 1 — GTSP equivalence with a new formulation

What they asked

Prove (or disprove) that SSP is equivalent to GTSP, using a non-compact formulation whose variables are padded magazine configurations — i.e., every feasible tool-set loaded to exactly capacity C.

What this means concretely
  • Enumerate all configurations (tool subsets of size exactly C that cover at least one job's requirement). These become columns/nodes in your formulation.
  • A job's tool requirements fit in configuration S if T_j ⊆ S. Each job thus corresponds to a cluster of feasible configurations.
  • Transitioning from configuration S to S' costs |S \ S'| tool switches. This is a travel cost on a complete directed graph of configurations.
  • You need to find a Hamiltonian path (or cycle) that visits exactly one configuration per job-cluster — this is exactly the GTSP.
  • The non-compact part: you should write an LP/IP formulation using configuration-selection variables and inter-configuration transition variables, not job-sequence variables.
  • Deliverable: a theorem stating the equivalence + the formulation written up formally + a small example (5–6 jobs, 8–10 tools, C=3 or 4).

Task 2 — JGP/TSIP investigation

What they asked

If you solve JGP optimally, will the resulting configurations necessarily be optimal for SSP? Find a counterexample or a proof.

What this means
  • JGP groups jobs into bins so that each bin's union of tool requirements fits in C slots. It minimises the number of bins (= number of distinct configurations, ignoring sequence).
  • SSP also sequences those bins to minimise transitions. JGP-optimal ≠ SSP-optimal in general because the grouping that needs fewest distinct configs may require many costly transitions.
  • Counterexample strategy: design a small instance where the JGP-optimal grouping forces a worst-case TSP tour over configurations, while a suboptimal grouping enables a much cheaper tour.
  • Then run TSP/Concorde on the set of configurations produced by JGP, and compare the resulting SSP cost to the true SSP optimum on small instances.

Task 3 — Polytope exploration with PORTA

What they asked

Use PORTA's traf on small JGP instances to enumerate all vertices of the LP relaxation, classify optimal vs. sub-optimal integer vertices, then apply TSP/Concorde heuristics over the resulting configuration sets.

Workflow
  • Write the JGP as an ILP (bin-packing + covering constraints). Feed H-representation to PORTA → get V-representation (all vertices).
  • Identify: (a) integral vertices = feasible JGP solutions, (b) fractional vertices = LP relaxation artifacts.
  • For each integral vertex (= a configuration set), run TSP or Concorde on the induced transition graph → get an SSP cost.
  • Plot: JGP cost (# configs) vs. SSP cost (# switches). Are better JGP solutions associated with better SSP solutions? What is the range of SSP costs for JGP-optimal solutions?
  • This gives you empirical insights into the SSP polytope structure — publishable as a computational study.

Chronological SOTA: SSP and allied problems

The field has progressed from pure heuristics → exact MILP → column generation → symmetry-aware formulations. Below is a curated timeline of the most cited/impactful works.

1988 — Foundations
Tang & Denardo — KTNS policy + first exact method
Formally introduced SSP. Proved that given a fixed job sequence, the optimal tool loading follows the Keep Tool Needed Soonest (KTNS) policy. This decouples sequencing from loading.
1994 — Complexity
Crama et al. — NP-hardness proof
Proved SSP is NP-hard even with unbounded magazine capacity, motivating the heuristic line. Also established connections to set-cover problems.
1994–2001 — Heuristic era
Tabu search, SA, greedy approaches
Balakrishnan & Vanderbeck (1999), Privault & Finke (1995), Hertz et al. — classical metaheuristics applied to SSP. Competitive on instances up to ~20 jobs.
2004 — Exact / Polyhedral
Laporte et al. — first competitive MILP + TSP connection
Position-indexed MILP formulation. Explicitly noted SSP resembles TSP over a state space. Benchmark instances created here remain standard today.
2015 — Tighter bounds
Catanzaro et al. — three tighter MIP models
Strengthened LP relaxation bounds, showed significant reduction in branch-and-bound nodes. Could not solve large instances due to variable count explosion.
2019 — State-of-the-art survey
Calmels — PhD + state-of-the-art review
Comprehensive survey of 30+ years. Introduces the JGP decomposition perspective. Proposes construction heuristic for generating good configurations for column generation — likely the PhD thesis your advisor mentioned.
2020 — Hybrid GA
Mecler, Subramanian, Vidal — Hybrid Genetic Search
Best published heuristic as of 2020. Permutation representation + KTNS decoder + diversity management. Significantly outperforms all prior heuristics. Computers & Operations Research.
2021 — Multi-commodity flow
da Silva et al. — SSPMF (multi-commodity flow)
Compact MIP using multi-commodity flow structure. Fastest known exact method at time of publication, exploiting network structure for tighter LP bounds.
2021 — Parallel machines
Calmels — SSP-NPM (non-identical parallel machines)
MILP + ILS for multiple machines with different capacities and setups. Opens door to multi-machine variants. EJOR.
2024 — Symmetry breaking
Akhundov & Ostrowski — JGSMF (job grouping + symmetry)
Job-group based MILP that explicitly handles the symmetry problem. Solves hard instances that SSPMF could not. Uses orbital branching ideas from unit commitment. EJOR 316(3). This is the direct predecessor your formulation must beat or extend.
2025 — Arc flow + NPM
Multiple groups — SSP-NPM arc flow + BRKG
Arc flow models derived from job-group representation; biased random-key genetic algorithm for NPM. Most recent SOTA as of 2025. IEEE TASE + COR.

Allied problems — what transfers

Generalized TSP (GTSP)

Noon & Bean (1993): reduction to ATSP. Fischetti et al. (1997): branch-and-cut. Karapetyan & Gutin: local search neighborhoods. All transferable to SSP via config-graph.

Bin Packing / Set Cover

JGP is a generalized bin-packing. Set-cover LP relaxations + column generation (Gilmore-Gomory) directly apply to generating configurations.

TSIP (Tool Switching Instants)

Determines when to switch (not which sequence). Configuration-based formulation overlaps with SSP. Branch-and-price was proposed in prior student thesis — use as baseline.

Vehicle Routing (VRP)

Branch-price-and-cut methodology from VRP is directly applicable. ESPPRC pricing subproblems may arise if time-windows are added to SSP.

SSP ↔ GTSP ↔ JGP ↔ TSIP: the formal connections

SSP → GTSP

Construction: Create a node for every feasible padded magazine configuration (tool subset of size exactly C). Group nodes by which job they can service: cluster C_j = {S : T_j ⊆ S}. Edge weight between S and S' = |S \ S'| (switches needed). GTSP solution = visit one node per cluster in a Hamiltonian cycle/path. This is exactly SSP.

Equivalence direction: SSP ≤_p GTSP (polynomial reduction). Since GTSP ≤_p ATSP (Noon-Bean), SSP ≤_p ATSP. The reverse: can every GTSP instance be cast as an SSP? Not in general — GTSP clusters may not have the covering structure. So it's a one-way reduction, not full equivalence, unless you restrict to GTSP instances with the covering structure.

Implication: Every GTSP algorithm applies to SSP. The GTSP literature on local search (Karapetyan-Gutin 2-opt/3-opt neighborhoods) transfers directly — this is an underexplored direction!

SSP → JGP relationship

JGP solves: partition jobs into groups (bins) so each bin's tool requirements fit in C slots, minimizing number of bins (= configurations). It ignores the sequence between bins.

Two-phase decomposition: Phase 1 = JGP (group jobs); Phase 2 = TSP/GTSP on the resulting configurations (sequence groups). This is NOT optimal in general for SSP.

Gap example: Consider jobs {A,B,C,D} with tools such that JGP-optimal creates 3 configs {S1, S2, S3} requiring many transitions, while a suboptimal 4-config grouping enables a tour with far fewer switches. Formally: JGP-optimal grouping ≠ SSP-friendly grouping. Your counterexample is constructing this gap.

Bound: Let OPT_JGP = # bins optimal. Then OPT_SSP ≥ OPT_JGP − 1 (trivially). But OPT_SSP can be much larger than (OPT_JGP − 1) × 1, because each configuration transition can cost up to C switches.

TSIP relationship

TSIP asks: given a fixed job sequence, when to switch tools? This is the loading subproblem. SSP asks: what sequence? TSIP is the inner problem; SSP is the outer problem (KTNS policy solves TSIP optimally for a fixed sequence). The configuration-based approach for TSIP uses the same notion of magazine states as configurations — directly related.

Non-compact formulation structure

Standard compact formulations use O(n²) job-sequence variables. The non-compact (Dantzig-Wolfe) reformulation uses configuration variables λ_S ∈ {0,1} for each configuration S and transition variables x_{SS'} for each inter-config arc — potentially exponential in number but with a strong LP relaxation and the ability to eliminate symmetry (same configurations = interchangeable, so grouping breaks it).

The pricing subproblem for column generation: find the most profitable configuration, i.e., a set S of size C that covers at least one uncovered job and minimizes reduced cost. This is a weighted set cover / knapsack subproblem — solvable in pseudo-polynomial time.
