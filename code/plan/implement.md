### Phase 2: Configuration Harvesting via SCIP (The JGP Branch-and-Price)

**Goal:** Implement Felipe's Column Generation to solve the JGP and extract the `CONV_Vertices` equivalent (the $\Omega_{Harvest}$ pool).

**Task 2.1: Restricted Master Problem (RMP)**

- **Code to Write:** Define the Set Covering RMP in `PySCIPOpt` where columns are tool configurations.

**Task 2.2: The Exact Pricing Subproblem (PSP)**

- **Code to Write:** Implement the Set Union Knapsack Problem to maximize collected dual variables minus 1. Provide this to SCIP via the `ObjPricer` callback.

**Task 2.3: Ryan-Foster Branching and Harvesting**

- **Code to Write:** Implement the Ryan-Foster `SAME` and `DIFFER` branching constraints on fractional job pairs.
- **Critical Harvesting Logic:** Override the node exploration callback so that _every_ column generated by the PSP with a negative reduced cost is appended to an external JSON/CSV file or Python list (`Omega_Harvest`).
- _Prompt to Agent:_ "Implement the SCIP `ObjBranchrule` using Ryan-Foster. Do not stop at the first integer solution. Force the exploration of fractional nodes using `DIFFER` to intentionally generate near-optimal/sub-optimal configurations. Log every generated configuration to `Omega_Harvest`."

### Phase 3: ML-Augmented Pricing Heuristic (MLPH)

**Goal:** Bypass the exact NP-hard pricing solver by predicting negative reduced-cost columns using Machine Learning.

**Task 3.1: Data Generation and Model Training**

- **Code to Write:** Generate a training dataset by running the exact CG (Phase 2) on small instances. Record the state: (Job-tool matrix, current RMP dual variables $\pi_i$) and the target: (Did a tool belong to the optimal configuration?).
- **Code to Write:** Train a Graph Convolutional Neural Network (GCNN) or a fast Support Vector Machine (SVM) on this data.

**Task 3.2: MLPH Injection into SCIP**

- **Code to Write:** Modify the `scip_redcost` callback.
- **Execution Flow:**
  1. Pass duals $\pi$ to the MLPH.
  2. MLPH outputs tool probabilities.
  3. Use probabilistic sampling to build $k$ candidate configurations.
  4. Calculate reduced cost ($1 - \sum \pi_i$). If $<0$, add to RMP.
  5. If all $k$ candidates fail, call the exact PSP solver (Task 2.2).
- _Prompt to Agent:_ "Write the `scip_redcost` callback to first query the trained ML model. Implement a fallback mechanism so mathematical exactness is preserved if the ML model hallucinates."

### Phase 4: Group Sequencing Problem (GSP) and Routing

**Goal:** Route the harvested discrete configurations to form a continuous schedule.

**Task 4.1: Subsetting and ATSP Matrix Generation**

- **Code to Write:** Create a function that searches `Omega_Harvest` for subsets $\mathcal{P}$ that perfectly cover the job set $J$.
- **Code to Write:** For each cover $\mathcal{P}$, construct an Asymmetric TSP (ATSP) distance matrix where $d(C_i, C_j) = |C_j \setminus C_i|$. Introduce the Magazine Setup Time parameter $S_{stop}$, such that $d(C_i, C_j) = S_{stop} + c_{insert} \cdot |C_j \setminus C_i|$.

**Task 4.2: Concorde Integration**

- **Code to Write:** Implement the Jonker-Volgenant transformation to convert the ATSP matrix into a Symmetric TSP matrix.
- **Code to Write:** Pipe the symmetric matrix into `pyconcorde` (or output a `.tsp` file for LKH). Return the optimally routed sequence of configurations.

### Phase 5: Boundary Dissolution via Neural-ALNS

**Goal:** Smooth the rigid boundaries of the JGP configurations using the continuous KTNS policy.

**Task 5.1: ALNS Initialization and Operators**

- **Code to Write:** Initialize the ALNS sequence with the optimal routing from Phase 4.
- **Code to Write (Boundary Destroy):** Implement an operator that exclusively identifies the edges of the JGP batches (where $C_k \to C_{k+1}$) and removes the $q$ jobs adjacent to these boundaries.
- **Code to Write (KTNS-Greedy Repair):** Reinsert the removed jobs evaluating the cost delta using the exact KTNS simulator from Task 1.1.

**Task 5.2: Q-Learning Operator Selection**

- **Code to Write:** Wrap the destroy/repair operator selection in a Q-Learning Credit Assignment (QLCA) loop. Update Q-values based on the reduction in total tool switches.
- _Prompt to Agent:_ "Construct an ALNS loop. The state space for Q-learning is the current optimality gap. The reward signal is the delta in the KTNS objective function. Use the Boundary Destroy and KTNS-Greedy Repair operators."
